# 个人实训总结

## 一、项目概述

本次实训我做的是一个全栈电商网站——"优选商城"，用户可以浏览商品、加入购物车、下单购买、查看历史订单。前端用 Next.js 实现，后端用 Flask 实现，数据库用 SQLite。

## 二、遇到的最大困难及解决过程

开发过程中最大的困难是前后端联调时的跨域问题。当时后端已经写好了，API 用 Postman 测试也都能正常返回数据，但前端页面一打开就报"加载商品失败"。我一开始以为是 API 地址写错了，检查了好几遍发现没问题。后来打开浏览器的开发者工具看 Network 面板，发现请求返回了 CORS 错误。

原因出在后端的 CORS 配置里只写了 `localhost:3000`，但 Next.js 启动的时候 3000 端口被占用了，自动换成了 3001 端口。我把 3001 加到 CORS 白名单后，页面立刻就正常显示了。这个 bug 花了我大概一个小时才定位到。

第二个困难是购物车的数量联动。AI 生成的代码里，购物车加减数量后不会自动更新页面上的角标数字。我最后用 `window.dispatchEvent` 派发自定义事件来通知 Header 组件刷新。

## 三、AI 辅助编程的心得体会

这次项目我全程使用了 TRAE IDE 的 AI 助手。说实话，AI 的代码生成能力确实很强，基本上你说清楚需求，它就能给你生成一套能跑的代码。项目骨架（目录结构、数据库模型、API 路由、前端页面）这些重复性高的工作，用 AI 来做效率非常高。

但也有明显的缺点。AI 生成的代码有时候"看着对但跑不了"，比如 CORS 那个问题。还有购物车结算的时候，AI 一开始没处理好下单后清空购物车的逻辑，是我自己看 API 文档才改对的。

我觉得用 AI 辅助编程的正确方式是：让 AI 搭框架、写基础代码，然后自己来调试、测试、完善细节。不能完全依赖 AI，也不能完全不用。

## 四、个人在项目中的具体贡献

这个项目是我独立完成的，AI 只是辅助工具。具体来说：

1. 我自己确定了项目方向（电商），设计了功能模块
2. 向 AI 描述需求，让它生成代码
3. 排查并修复了 CORS 跨域问题
4. 调整了购物车数量联动的逻辑
5. 补充了订单页面的结算功能
6. 修改了 API 文档中与实际不符的部分
7. 手动配置了 Git 仓库并推送到 GitHub

## 五、总结

通过这次实训，我对全栈开发有了更直观的理解，特别是前后端如何通过 API 通信、数据库如何设计表之间的关系。同时也体会到了 AI 辅助编程的利与弊。AI 能帮你快速起步，但真正的能力还是在于你能理解代码、能排查问题、能把项目做得完整。
